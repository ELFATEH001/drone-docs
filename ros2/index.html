    ROS 2 Stack

    Overview

    The ROS 2 stack runs entirely onboard the Raspberry Pi 4 companion computer, which is mounted on the drone frame and flies with it. The ground laptop is used for monitoring only — all control computation happens on the RPi.

    Topic Graph

    Publisher Topic Subscriber
    v4l2_camera /camera/image_raw aruco_detector_node
    aruco_detector_node /aruco/marker_centroid visual_servo_node
    mavros /mavros/local_position/odom visual_servo_node
    visual_servo_node /mavros/setpoint_raw/local mavros

    Components

    MAVROS

    MAVROS is the translation bridge between ROS 2 and ArduPilot via MAVLink over UART.

    Key topics:

    Topic Type Direction
    /mavros/state mavros_msgs/State Read — connection status
    /mavros/local_position/odom nav_msgs/Odometry Read — pose + velocity
    /mavros/setpoint_raw/attitude mavros_msgs/AttitudeTarget Write — thrust + quaternion
    /mavros/setpoint_raw/local mavros_msgs/PositionTarget Write — velocity setpoints

    Launch: 

    ros2 launch mavros apm.launch fcu_url:=/dev/ttyAMA0:921600

    Verify connection: 

    ros2 topic echo /mavros/state
# connected: True

    Camera Driver

    The downward-facing camera is exposed via v4l2_camera (USB) or raspicam2 (Pi Camera). Recommended resolution: 640×480 @ 30 FPS.

    Calibrate with a checkerboard: 

    ros2 run camera_calibration cameracalibrator \
  --size 8x6 --square 0.025 \
  --ros-args -r image:=/camera/image_raw

    Target reprojection error: < 0.5 px. The resulting camera_info.yaml provides the focal length \(f\) needed for the pixel-to-metric conversion.

    ArUco Detector

    The ros2_aruco package detects markers and publishes their centroid and pose.

    Parameter Value
    Dictionary DICT_4X4_50
    Primary marker size 40 cm
    Target detection rate ≥ 15 FPS on RPi

    SE3 Controller Node

    The geometric SE(3) controller runs as a C++ ROS 2 node (controller_node_visual) at 100 Hz using Eigen for all linear algebra.

    Subscriptions:

    Topic Usage
    /mavros/local_position/odom Position \(p^w\), velocity \(v^w\)
    /mavros/local_position/pose Orientation \(R^w_B\)
    /aruco/marker_centroid Visual error \((e_u, e_v)\)

    Publications:

    Topic Usage
    /mavros/setpoint_raw/attitude Thrust \(f^B_z\) + attitude quaternion
    /se3/debug Debug array for PlotJuggler

    Tunable parameters:

    Parameter Default Description
    kp 1.0 Position gain
    kv 2.0 Velocity damping
    kR 1.5 Attitude gain
    kOmega 0.5 Angular velocity damping
    vmax 0.3 Max velocity command (m/s)

    Visual Servo Node

    Converts the ArUco pixel error into metric velocity commands:

    \[e_u = u_{\text{marker}} - u_0, \quad e_v = v_{\text{marker}} - v_0\]
    \[e_x = \frac{h}{f} e_u, \quad e_y = \frac{h}{f} e_v\]
    \[v^{\text{cmd}}_x = \text{clip}(k_x\, e_x,\ -v_{\text{max}},\ +v_{\text{max}})\]
    \[v^{\text{cmd}}_y = \text{clip}(k_y\, e_y,\ -v_{\text{max}},\ +v_{\text{max}})\]

    where \(h\) is altitude from the barometer and \(f\) is the calibrated focal length.

    If no marker is detected for more than 0.5 s, the node holds the last setpoint and publishes a warning on /rosout.

    Debugging

    Monitor all signals live with PlotJuggler: 

    ros2 run plotjuggler plotjuggler

    Check node health: 

    ros2 node list
ros2 topic hz /mavros/local_position/odom
ros2 topic hz /se3/debug
